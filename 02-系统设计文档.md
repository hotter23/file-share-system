# 系统设计文档

## 1. 系统架构设计

### 1.1 整体架构

本系统采用 Spring Cloud 微服务架构，前端 Vue.js 通过 Nginx 反向代理访问后端 API 网关，后端各微服务通过 Docker Compose 容器化部署，文件存储使用腾讯云 COS。

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户浏览器                               │
│                    Vue.js + Element Plus                        │
└──────────────────────────────────┬──────────────────────────────┘
                                   │ HTTPS
┌──────────────────────────────────▼──────────────────────────────┐
│                         Nginx 服务器                            │
│                    反向代理 + 静态资源服务                       │
└──────────────────────────────────┬──────────────────────────────┘
                                   │ HTTP
┌──────────────────────────────────▼──────────────────────────────┐
│                    Spring Cloud Gateway                         │
│                        API 网关 (:8000)                          │
│                  路由转发 / 认证鉴权 / 限流熔断                    │
└────┬─────────────────┬─────────────────┬─────────────────┬────────┘
     │                 │                 │                 │
┌────▼────┐      ┌─────▼────┐     ┌─────▼────┐     ┌─────▼────┐
│用户服务  │      │文件服务   │     │分享服务   │     │存储服务   │
│:8081    │      │:8082    │     │:8083    │     │:8084    │

┌────▼─────────────────▼─────────────────▼─────────────────▼────────┐
│                         MySQL 数据库                             │
│              用户数据 / 文件元数据 / 分享数据                     │
├─────────────────────────────────────────────────────────────────┤
│                         Redis 缓存                               │
│                  Token 缓存 / 会话管理 / 热点数据                  │
├─────────────────────────────────────────────────────────────────┤
│                    Docker 容器网络                               │
└─────────────────────────────────────────────────────────────────┘
                                   │
                    ┌──────────────▼──────────────┐
```

### 1.2 微服务设计

#### 1.2.1 API 网关 (Gateway Service)
- **端口**: 8000
- **职责**:
  - 统一路由：所有前端请求经过网关分发到对应微服务
  - 认证鉴权：校验 JWT Token
  - 限流熔断：保护后端服务
  - 跨域处理：统一处理 CORS

#### 1.2.2 用户服务 (User Service)
- **端口**: 8081
- **职责**:
  - 用户注册、登录、登出
  - JWT Token 发放与刷新
  - 用户信息管理
  - 管理员用户管理

#### 1.2.3 文件服务 (File Service)
- **端口**: 8082
- **职责**:
  - 文件上传（简单上传、分片上传）
  - 文件下载
  - 文件管理（列表、搜索、删除、重命名）
  - 文件夹管理

#### 1.2.4 分享服务 (Share Service)
- **端口**: 8083
- **职责**:
  - 生成分享链接
  - 分享链接访问
  - 分享权限验证
  - 分享管理

#### 1.2.5 存储服务 (COS Service)
- **端口**: 8084
- **职责**:
  - 封装腾讯云 COS SDK
  - 文件上传至 COS
  - 文件下载自 COS
  - 分片上传管理
  - 文件删除

### 1.3 Docker 容器化设计

每个微服务打包为独立的 Docker 容器，通过 Docker Compose 编排。

| 容器名 | 服务 | 端口映射 |
|--------|------|----------|
| gateway | API 网关 | 8000:8000 |
| user-service | 用户服务 | 8081:8081 |
| file-service | 文件服务 | 8082:8082 |
| share-service | 分享服务 | 8083:8083 |
| cos-service | 存储服务 | 8084:8084 |
| mysql | 数据库 | 3306:3306 |
| redis | 缓存 | 6379:6379 |
| nginx | 反向代理 | 80:80 |

## 2. 模块设计

### 2.1 用户认证模块

#### 2.1.1 认证流程

```
用户登录
    │
    ▼
┌─────────┐    用户名/密码    ┌──────────────┐
│ 前端表单 │ ──────────────▶  │  User Service │
└─────────┘                   └───────┬──────┘
                                        │
                                        ▼
                                ┌──────────────┐
                                │ 验证用户名密码 │
                                │ (BCrypt)     │
                                └───────┬──────┘
                                        │
                                        ▼
                                ┌──────────────┐
                                │ 生成 JWT Token │
                                │ (24h有效期)   │
                                └───────┬──────┘
                                        │
                                        ▼
                                返回 Token 给前端
                                        │
                                        ▼
                                前端存储 Token
                                后续请求携带 Token
```

#### 2.1.2 JWT Token 结构

```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "userId",
    "username": "string",
    "role": "USER|ADMIN",
    "iat": 1713344400,
    "exp": 1713430800
  }
}
```

### 2.2 文件上传模块

#### 2.2.1 简单上传流程（小文件 < 100MB）

```
前端选择文件
    │
    ▼
前端校验文件类型/大小
    │
    ▼
FormData 提交到 /api/file/upload
    │
    ▼
File Service 接收文件流
    │
    ├─── 文件信息写入 MySQL (status=PENDING)
    │
    ▼
调用 COS Service 上传到 COS
    │
    ▼
COS 返回 cosKey
    │
    ▼
更新 MySQL 记录 (status=COMPLETED)
    │
    ▼
返回上传结果给前端
```

#### 2.2.2 分片上传流程（大文件 ≥ 100MB）

```
前端选择文件
    │
    ▼
前端计算文件 MD5
    │
    ▼
请求分片上传初始化 /api/file/upload/init
    │
    ▼
File Service 返回 fileUuid, uploadId, chunkCount
    │
    ▼
前端按分片并行上传 (每片 5MB)
    │
    ▼
所有分片上传完成
    │
    ▼
请求分片上传完成 /api/file/upload/complete
    │
    ▼
COS Service 合并分片
    │
    ▼
更新 MySQL (status=COMPLETED)
    │
    ▼
返回上传结果
```

#### 2.2.3 断点续传设计

- 每次分片上传前检查该分片是否已上传（通过 file_chunk 表）
- 若分片已存在，跳过该分片
- 若网络中断，重试时从断点继续

### 2.3 文件下载模块

#### 2.3.1 普通下载

```
用户请求下载
    │
    ▼
验证用户权限（文件所有者）
    │
    ▼
从 COS 获取文件流
    │
    ▼
返回文件流，设置 Content-Disposition
```

#### 2.3.2 分享下载

```
用户访问分享链接
    │
    ▼
验证分享是否过期
    │
    ▼
验证分享密码（如有）
    │
    ▼
更新分享访问次数
    │
    ▼
获取文件，返回下载
```

### 2.4 权限控制设计

#### 2.4.1 认证层级

## 3. 存储设计

### 3.1 COS 存储路径规范

```
bucket/
└── users/
    └── {userId}/
        └── {year}/
            └── {month}/
                └── {fileUuid}/
                    └── {fileName}
```

**示例**:
```
file-share-bucket/
└── users/
    └── 10001/
        └── 2026/
            └── 04/
                └── abc123/
                    └── document.pdf
```

### 3.2 文件命名策略

- **业务 ID**: 使用 UUID 作为文件业务主键
- **COS 存储名**: `{userId}/{date}/{uuid}/{originalName}`
- **分片命名**: `{uploadId}/chunk_{index}`

### 3.3 缓存策略

| 数据类型 | 缓存位置 | TTL | 说明 |
|----------|---------|-----|------|
| JWT Token | Redis | 24h | 快速验证 |
| 用户信息 | Redis | 1h | 减少 DB 查询 |
| 分享信息 | Redis | 10min | 热点分享加速 |
| 文件元数据 | Redis | 30min | 列表查询加速 |

## 4. 安全设计

### 4.1 传输安全

- 所有 API 使用 HTTPS 加密传输
- 前端与后端通信强制 HTTPS
- Nginx 配置 TLS 1.2+

### 4.2 认证安全

- 密码使用 BCrypt 加密（强度 10）
- JWT Token 设置合理有效期（24h）
- Token 刷新机制实现无感续期
- 敏感接口添加防暴力破解限制

### 4.3 数据安全

- COS 服务端 AES-256 加密
- 重要配置使用环境变量而非硬编码
- SQL 参数化查询防止注入
- XSS 过滤用户输入

### 5.3 Docker Compose 编排

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: file_share
    volumes:
      - mysql_data:/var/lib/mysql
      - ./docker/mysql/init.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "3306:3306"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  gateway:
    build: ./backend/file-share-gateway
    ports:
      - "8000:8000"
    depends_on:
      - user-service
      - file-service
      - share-service

  user-service:
    build: ./backend/file-share-user
    ports:
      - "8081:8081"
    environment:
      SPRING_REDIS_HOST: redis
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/file_share

  file-service:
    build: ./backend/file-share-file
    ports:
      - "8082:8082"

  share-service:
    build: ./backend/file-share-share
    ports:
      - "8083:8083"

  cos-service:
    build: ./backend/file-share-cos
    ports:
      - "8084:8084"

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./docker/nginx/nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - gateway

volumes:
  mysql_data:
```

## 7. 错误处理

### 7.1 全局异常处理

| 异常类型 | HTTP 状态码 | 错误码 | 说明 |
|----------|-------------|--------|------|
| BadRequestException | 400 | 400 | 请求参数错误 |
| UnauthorizedException | 401 | 401 | 未认证 |
| ForbiddenException | 403 | 403 | 无权限 |
| ResourceNotFoundException | 404 | 404 | 资源不存在 |
| FileTooLargeException | 413 | 413 | 文件过大 |
| UnsupportedMediaTypeException | 415 | 415 | 不支持的文件类型 |
| ServiceException | 500 | 500 | 服务内部错误 |

### 7.2 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1713344400000,
  "path": "/api/file/upload"
}
```

错误响应：
```json
{
  "code": 404,
  "message": "文件不存在",
  "data": null,
  "timestamp": 1713344400000,
  "path": "/api/file/download/xxx"
}
```
